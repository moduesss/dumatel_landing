# Руководство по работе с панелью управления блогом (Sanity)

Данное руководство предназначено для авторов и редакторов, которые будут наполнять блог: писать статьи, управлять разделами и авторами, а также понимать ограничения и логику работы панели. Ниже подробно описаны все доступные типы данных и правила их заполнения.

---

## Публикации

Раздел **«Публикация»** используется для создания и редактирования статей блога. Каждая публикация — это отдельная статья, которая отображается на сайте.

### Заголовок

Поле **«Заголовок»** — это основной заголовок статьи, который видит пользователь на сайте и в списке публикаций. Заголовок должен быть осмысленным, отражать суть материала и не содержать служебных символов. Поле является обязательным — без него публикацию сохранить невозможно.

Важно учитывать, что именно на основе заголовка автоматически формируется ссылка на статью.

### Ссылка

Поле **«Ссылка»** (slug) — это URL статьи. По умолчанию оно автоматически создаётся из заголовка и приводится к человеко-читаемому виду (латиница, дефисы).

Ссылку можно отредактировать вручную, если требуется более короткий или SEO-оптимизированный вариант. Максимальная длина ссылки — 96 символов. Поле обязательно для заполнения.

> ⚠️ Рекомендация: после публикации статьи не изменяйте ссылку, так как это приведёт к изменению URL и может нарушить уже существующие ссылки на материал.

### Анонс

Поле **«Анонс»** используется для краткого описания статьи в списке публикаций, карточках и превью. Обычно это 2–4 предложения, которые дают общее понимание содержания материала и мотивируют перейти к чтению.

Поле обязательное. Не рекомендуется копировать первые абзацы статьи — лучше сформулировать отдельный, ёмкий текст.

### SEO-ключевые слова

Поле **«SEO-ключевые слова»** предназначено для указания ключевых слов и фраз, по которым статья может быть релевантна в поисковых системах.

Заполнение происходит в формате тегов: каждое ключевое слово или фраза добавляется отдельно. Поле необязательное, но рекомендуется для SEO-оптимизации. Жёстких ограничений по количеству или длине нет, однако не стоит злоупотреблять — достаточно 5–10 релевантных ключей.

### Автор

Поле **«Автор»** связывает публикацию с конкретным автором из справочника. Автор выбирается из заранее созданных записей в разделе **«Автор»**.

Поле обязательное. Если нужного автора нет в списке, его необходимо сначала создать в соответствующем разделе.

### Разделы

Поле **«Разделы»** используется для привязки статьи к одной или нескольким рубрикам (категориям). Разделы выбираются из справочника **«Раздел»**.

Поле необязательное, но крайне желательно для корректной навигации и фильтрации контента на сайте. Один и тот же раздел нельзя добавить несколько раз — система автоматически следит за уникальностью.

### Дата публикации

Поле **«Дата публикации»** является **обязательным для заполнения**. Оно определяет, с какого момента статья считается опубликованной и в каком порядке материалы отображаются на сайте.

В это поле можно указать:
- текущую дату и время — для немедленной публикации;
- будущую дату — если необходимо запланировать публикацию заранее.

Таким образом, система поддерживает **отложенную публикацию**: статья может быть подготовлена заранее и автоматически появиться на сайте в указанную дату и время.

### Текст статьи

Поле **«Текст статьи»** — это основной контент. Здесь размещается полный текст материала с возможностью использовать заголовки, абзацы, списки, цитаты и изображения.

Рекомендуется структурировать текст: использовать подзаголовки, логические блоки и визуальные разделения для удобства чтения.

---

## Разделы (Рубрики)

Раздел **«Раздел»** используется для управления рубриками блога. Разделы помогают группировать статьи по темам.

### Название

Поле **«Название»** — это имя раздела, которое отображается на сайте. Оно должно быть коротким и понятным. Поле обязательное.

### Ссылка

Поле **«Ссылка»** формируется автоматически из названия раздела и используется в URL страниц разделов. Максимальная длина — 96 символов. Поле обязательное.

### Описание

Поле **«Описание»** — это краткое текстовое пояснение, что представляет собой данный раздел. Может использоваться для SEO или отображения на странице раздела. Поле необязательное.

---

## Авторы

Раздел **«Автор»** содержит информацию об авторах статей.

### Имя

Поле **«Имя»** — это имя автора, которое отображается на сайте. Поле обязательное.

### Ссылка

Поле **«Ссылка»** формируется автоматически из имени автора и используется в URL страницы автора. Максимальная длина — 96 символов. Поле обязательное.

### Должность

Поле **«Должность»** необязательное. Здесь можно указать роль автора, например: «Редактор», «Маркетолог», «Эксперт компании».

### Фото

Поле **«Фото»** используется для загрузки аватара автора. Изображение необязательное, но рекомендуется для улучшения визуального восприятия контента. Поддерживается обрезка (hotspot).

---

## Настройки сайта

Раздел **«Настройки»** содержит глобальные параметры сайта, которые используются для SEO и превью.

### Название сайта

Поле **«Название сайта»** используется в заголовках страниц и мета-тегах. Рекомендуется заполнять официальным названием проекта.

### Описание сайта

Поле **«Описание сайта»** — это краткое описание ресурса, используемое для SEO и превью в поисковых системах и соцсетях.

### OG-изображение

Поле **«OG-изображение»** задаёт изображение по умолчанию для предпросмотра ссылок при шаринге. Поддерживается кадрирование (hotspot). Поле необязательное, но рекомендуется.

---

## Общие рекомендации

- Всегда проверяйте обязательные поля перед сохранением.
- Не меняйте ссылки (slug) у уже опубликованных материалов без необходимости.
- Старайтесь писать понятные заголовки и анонсы — они напрямую влияют на кликабельность.
- Используйте разделы и авторов последовательно, чтобы структура блога оставалась логичной.

При соблюдении этих правил работа с панелью будет простой, а контент — аккуратным и структурированным.
